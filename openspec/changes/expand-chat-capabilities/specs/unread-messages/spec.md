## ADDED Requirements

### Requirement: 채팅방별 읽음 위치 저장
시스템은 인증 사용자의 마지막 읽음 메시지 위치를 채팅방별로 MUST 저장해야 한다.

#### Scenario: 채팅방 메시지 확인
- **WHEN** 사용자가 채팅방의 최신 메시지를 확인한다
- **THEN** 시스템은 해당 사용자와 채팅방의 마지막 읽음 시간 및 메시지 ID를 저장한다

#### Scenario: 늦게 도착한 읽음 요청
- **WHEN** 최신 읽음 요청 이후 더 오래된 메시지의 읽음 요청이 도착한다
- **THEN** 시스템은 저장된 읽음 위치를 과거로 변경하지 않는다

#### Scenario: 다른 사용자의 읽음 위치 조회
- **WHEN** 인증 사용자가 다른 사용자의 읽음 위치 행을 조회한다
- **THEN** 데이터베이스 정책은 요청을 거부한다

### Requirement: 안 읽은 메시지 구분 표시
시스템은 채팅방 재입장 시 이전 읽음 위치 이후의 다른 사용자 메시지를 MUST 구분해 표시해야 한다.

#### Scenario: 안 읽은 메시지가 있는 재입장
- **WHEN** 사용자가 이전 읽음 위치 이후 새 메시지가 있는 채팅방을 다시 연다
- **THEN** 시스템은 첫 안 읽은 메시지 앞에 `새 메시지 N개` 구분선을 표시한다

#### Scenario: 본인이 보낸 메시지
- **WHEN** 이전 읽음 위치 이후 메시지 중 발신자가 현재 사용자다
- **THEN** 시스템은 해당 메시지를 안 읽은 개수에 포함하지 않는다

#### Scenario: 처음 입장한 채팅방
- **WHEN** 사용자에게 저장된 읽음 위치가 없다
- **THEN** 시스템은 기존 전체 내역을 새 메시지로 표시하지 않고 최신 위치를 저장한다

#### Scenario: 현재 화면의 실시간 메시지
- **WHEN** 사용자가 보고 있는 채팅방에 새 메시지가 실시간 도착한다
- **THEN** 시스템은 메시지를 표시하고 최신 읽음 위치를 갱신한다

### Requirement: 초기 메시지 포커스
시스템은 채팅방 로딩 완료 시 사용자가 읽기를 시작할 위치로 화면을 MUST 이동해야 한다.

#### Scenario: 안 읽은 메시지가 있는 로딩
- **WHEN** 안 읽은 메시지가 있는 채팅방 로딩이 완료된다
- **THEN** 시스템은 첫 안 읽은 메시지와 새 메시지 구분선이 보이는 위치로 스크롤한다

#### Scenario: 안 읽은 메시지가 없는 로딩
- **WHEN** 안 읽은 메시지가 없는 채팅방 로딩이 완료된다
- **THEN** 시스템은 마지막 메시지가 보이는 위치로 스크롤한다

#### Scenario: 로딩 후 실시간 메시지 수신
- **WHEN** 사용자가 최신 메시지 근처를 보고 있을 때 새 메시지가 실시간 도착한다
- **THEN** 시스템은 새 메시지를 표시하고 마지막 메시지가 보이는 위치로 스크롤한다

#### Scenario: 이전 대화를 읽는 중 실시간 메시지 수신
- **WHEN** 사용자가 최신 위치에서 벗어나 이전 대화를 읽는 중 새 메시지가 도착한다
- **THEN** 시스템은 현재 스크롤 위치를 유지하고 새 메시지 알림과 최신 위치로 이동하는 동작을 표시한다
