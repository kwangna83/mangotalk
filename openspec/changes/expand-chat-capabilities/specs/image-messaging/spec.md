## ADDED Requirements

### Requirement: 이미지 선택과 검증
시스템은 참여자가 지원되는 이미지 한 개를 선택하고 전송 전에 미리보기와 유효성 결과를 MUST 확인할 수 있게 해야 한다.

#### Scenario: 지원 이미지 선택
- **WHEN** 참여자가 10MB 이하 JPEG, PNG 또는 WebP 이미지를 선택한다
- **THEN** 시스템은 미리보기와 전송·취소 동작을 표시한다

#### Scenario: 지원하지 않는 이미지
- **WHEN** 참여자가 허용되지 않은 MIME type이거나 decode할 수 없는 파일을 선택한다
- **THEN** 시스템은 Storage 요청 없이 형식 오류를 표시한다

#### Scenario: 크기 제한 초과
- **WHEN** 참여자가 10MB를 초과하는 이미지를 선택한다
- **THEN** 시스템은 Storage 요청 없이 크기 제한을 표시한다

### Requirement: 비공개 이미지 저장
시스템은 이미지 객체를 비공개 Storage에 저장하고 해당 방 참여자만 읽을 수 있도록 MUST 제한해야 한다.

#### Scenario: 참여자의 이미지 업로드
- **WHEN** 참여자가 현재 방에 유효한 이미지를 전송한다
- **THEN** 시스템은 방과 발신자 ID가 포함된 본인 전용 경로에 객체를 저장한다

#### Scenario: 비참여자의 이미지 조회
- **WHEN** 방에 참여하지 않은 사용자가 해당 방의 이미지 객체 또는 signed URL을 요청한다
- **THEN** Storage 정책은 요청을 거부한다

#### Scenario: 다른 사용자 경로 위조
- **WHEN** 참여자가 다른 사용자의 ID가 포함된 경로에 객체를 업로드한다
- **THEN** Storage 정책은 요청을 거부한다

### Requirement: 이미지 메시지 확정
시스템은 업로드된 객체를 이미지 메시지와 첨부 메타데이터에 연결하여 멱등하게 MUST 저장해야 한다.

#### Scenario: 이미지 업로드 완료
- **WHEN** Storage 업로드가 성공한다
- **THEN** 시스템은 메시지와 첨부 메타데이터를 원자 저장하고 전송 성공 상태를 표시한다

#### Scenario: 저장 응답 유실 후 재시도
- **WHEN** 동일한 클라이언트 메시지 ID로 이미지 메시지 저장을 재시도한다
- **THEN** 시스템은 메시지와 첨부를 중복 생성하지 않고 기존 결과를 반환한다

#### Scenario: 메타데이터 저장 실패
- **WHEN** 객체 업로드 후 메시지 확정이 실패한다
- **THEN** 시스템은 실패 상태와 재시도를 제공하고 참조되지 않은 객체 삭제를 시도한다

### Requirement: 이미지 메시지 표시
시스템은 이미지 메시지를 실시간 수신과 이전 내역 조회에서 중복 없이 MUST 표시해야 한다.

#### Scenario: 새 이미지 메시지 수신
- **WHEN** 현재 방의 참여자가 이미지 메시지를 저장한다
- **THEN** 연결된 사용자는 새로고침 없이 이미지 placeholder와 최종 이미지를 한 번 확인한다

#### Scenario: 만료된 signed URL
- **WHEN** 표시 중인 이미지 URL이 만료된다
- **THEN** 시스템은 참여 권한을 다시 확인해 URL을 갱신하고 이미지를 다시 로드한다

#### Scenario: 이미지 로딩 실패
- **WHEN** 이미지 다운로드 또는 decode가 실패한다
- **THEN** 시스템은 레이아웃을 유지한 오류 placeholder와 재시도 동작을 표시한다

### Requirement: 이미지 업로드 상태와 접근성
시스템은 업로드 진행·실패 상태와 이미지의 접근 가능한 설명을 MUST 제공해야 한다.

#### Scenario: 업로드 진행 중
- **WHEN** 이미지 바이트가 Storage로 전송 중이다
- **THEN** 시스템은 진행 상태를 표시하고 같은 이미지를 중복 전송하지 않는다

#### Scenario: 화면 읽기 사용
- **WHEN** 화면 읽기 사용자가 이미지 메시지에 초점을 이동한다
- **THEN** 시스템은 발신자와 이미지 메시지임을 설명하고 사용 가능한 재시도 동작을 안내한다
