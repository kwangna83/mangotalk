## ADDED Requirements

### Requirement: 익명 사용자 세션 생성
시스템은 유효한 닉네임을 제출한 사용자에 대해 Supabase Auth 익명 세션을 생성하고, 인증 사용자 ID에 연결된 프로필을 MUST 생성해야 한다.

#### Scenario: 유효한 닉네임으로 시작
- **WHEN** 인증되지 않은 사용자가 앞뒤 공백을 제외한 2자 이상 20자 이하의 닉네임을 제출한다
- **THEN** 시스템은 익명 인증 세션과 해당 사용자 ID의 프로필을 생성하고 공용 채팅 화면으로 이동한다

#### Scenario: 이미 인증된 사용자가 다시 시작
- **WHEN** 유효한 익명 세션이 남아 있는 사용자가 앱을 실행한다
- **THEN** 시스템은 새 사용자를 만들지 않고 기존 세션과 프로필을 사용한다

### Requirement: 닉네임 검증
시스템은 닉네임의 앞뒤 공백을 제거하고 길이를 검증하며, 유효하지 않은 값으로 인증 또는 프로필 요청을 보내지 MUST 않아야 한다.

#### Scenario: 빈 닉네임 거부
- **WHEN** 사용자가 비어 있거나 공백만 있는 닉네임을 제출한다
- **THEN** 시스템은 입력 오류를 표시하고 세션을 생성하지 않는다

#### Scenario: 허용 길이를 벗어난 닉네임 거부
- **WHEN** 정리된 닉네임 길이가 2자 미만이거나 20자를 초과한다
- **THEN** 시스템은 허용 길이를 안내하고 세션을 생성하지 않는다

### Requirement: 세션 복원과 만료 처리
시스템은 앱 재실행 시 저장된 인증 세션을 MUST 복원하고, 복원할 수 없는 세션은 안전하게 폐기해야 한다.

#### Scenario: 저장된 세션 복원
- **WHEN** 사용자가 유효한 세션을 가진 상태에서 앱을 다시 실행한다
- **THEN** 시스템은 로그인 입력을 다시 요구하지 않고 기존 프로필로 채팅 화면을 연다

#### Scenario: 만료되거나 유효하지 않은 세션
- **WHEN** 저장된 세션을 갱신하거나 검증할 수 없다
- **THEN** 시스템은 로컬 세션을 정리하고 닉네임 입력 화면을 표시한다

### Requirement: 프로필 접근 제어
시스템은 인증된 사용자가 자신의 프로필만 생성하거나 변경할 수 있도록 데이터베이스 정책으로 MUST 제한해야 한다.

#### Scenario: 다른 사용자 프로필 변경 차단
- **WHEN** 인증 사용자가 다른 사용자 ID의 프로필을 생성하거나 변경하려 한다
- **THEN** 데이터베이스는 해당 요청을 거부한다

### Requirement: 인증 오류 표시와 재시도
시스템은 인증 또는 프로필 생성 실패 시 사용자에게 오류와 재시도 수단을 MUST 제공해야 한다.

#### Scenario: 네트워크 오류로 시작 실패
- **WHEN** 세션 또는 프로필 생성 요청이 네트워크 오류로 실패한다
- **THEN** 시스템은 입력한 닉네임을 유지하고 실패 메시지와 재시도 동작을 표시한다
